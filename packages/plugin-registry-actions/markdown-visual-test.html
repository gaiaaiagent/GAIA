Key Findings

1. Blank Lines Are Required for Block Separation
According to the CommonMark specification, lists MUST be preceded by a blank line (unless at document start). Without it, the markdown parser treats the list markers (*) as literal text within the paragraph.
2. The space-y-0 Class Amplifies the Problem
Even if lists parse correctly, !space-y-0 removes all margins between elements. This means proper block structure is CRITICAL - you can't rely on CSS spacing to fix bad markdown.
3. Two-Space Line Breaks Don't Help
Adding two spaces at the end of a line creates a <br> tag, but this doesn't create block-level separation. Lists still won't parse correctly.
4. HTML <br> Tags Break Portability
While <br> tags technically work in Streamdown, they make the markdown non-portable (won't render correctly on GitHub, in Slack, etc.) and reduce accessibility.
5. Solution: Add Blank Lines Between ALL Block Elements
The fix is simple and spec-compliant: Add a blank line before every list, after every list, and between any block-level elements (paragraphs, headings, lists).

Implementation Guide

  • Step 1: Identify all block elements (headings, paragraphs, lists)
  • Step 2: Ensure blank line before and after each list
  • Step 3: Test in browser dev tools - verify <ul> elements exist
  • Step 4: Optionally use the MarkdownBuilder utility class for automatic spacing
  • Step 5: Copy/paste to GitHub markdown preview to verify portability
