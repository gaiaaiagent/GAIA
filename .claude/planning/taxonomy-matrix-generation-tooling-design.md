# Taxonomy Matrix Generation Tooling Design

## Vision

A progressive, semi-automated system that helps generate and maintain a comprehensive relationship matrix between project files.

## Core Architecture

### 1. File Analysis Engine

```typescript
interface FileMetadata {
  path: string;
  type: 'code' | 'doc' | 'config' | 'data';
  language?: string;
  imports?: string[];
  exports?: string[];
  references?: string[];
  lastModified: Date;
  size: number;
  complexity?: number;
  koi?: {
    rid: string;
    confidence: string;
  };
}
```

### 2. Relationship Detection System

#### Static Analysis (Automated)

- **Import/Export Graph**: AST parsing for TypeScript/JavaScript
- **File References**: Regex scanning for file paths
- **Configuration Dependencies**: Parse JSON/YAML references
- **Documentation Links**: Markdown link extraction

#### Semantic Analysis (AI-Assisted)

- **Conceptual Similarity**: Embed file contents, calculate cosine similarity
- **Shared Terminology**: Extract key terms, find overlaps
- **Pattern Recognition**: Identify similar code structures
- **Purpose Alignment**: Detect files serving similar goals

#### Dynamic Analysis (Runtime)

- **Execution Flow**: Trace which files are loaded together
- **API Call Patterns**: Monitor inter-module communication
- **Database Queries**: Track which models are used by which services

### 3. Progressive Documentation Pipeline

```typescript
interface RelationshipCell {
  from: string; // row file
  to: string; // column file
  strength: number; // 0-10
  types: RelationshipType[];
  autoGenerated: {
    imports: boolean;
    references: boolean;
    similarity: number;
    sharedTerms: string[];
  };
  humanReview: {
    status: 'pending' | 'reviewed' | 'approved';
    lastReviewed?: Date;
    reviewer?: string;
  };
  content?: {
    semantic: string;
    cognitive: string;
    implementation: string;
    metadata: Record<string, any>;
  };
}
```

## Implementation Strategy

### Step 1: Basic Scanner (Week 1)

```javascript
// matrix-scanner.js
export async function scanProject(rootPath) {
  const files = await inventoryFiles(rootPath);
  const relationships = await detectRelationships(files);
  return buildInitialMatrix(files, relationships);
}
```

### Step 2: Relationship Analyzer (Week 2)

```javascript
// relationship-analyzer.js
export async function analyzeRelationship(fileA, fileB) {
  const strength = calculateStrength(fileA, fileB);
  const types = detectRelationshipTypes(fileA, fileB);
  const sharedConcepts = findSharedConcepts(fileA, fileB);
  return { strength, types, sharedConcepts };
}
```

### Step 3: Content Generator (Week 3)

```javascript
// content-generator.js
export async function generateCellContent(relationship) {
  if (relationship.strength < 5) return null;

  const content = await buildAnalysis(relationship);
  return formatAsThreeParagraphs(content);
}
```

### Step 4: Review Interface (Week 4)

```javascript
// review-interface.js
export function createReviewDashboard(matrix) {
  const pending = matrix.filter((cell) => cell.humanReview.status === 'pending');
  return renderDashboard(pending);
}
```

This relationship exemplifies the TypeScript pattern of separating interface definitions from implementation, where types/index.ts acts as the contract that runtime.ts fulfills. Understanding types/index.ts is prerequisite to comprehending runtime.ts, as it establishes the mental model of agents, memories, messages, and their interactions.
