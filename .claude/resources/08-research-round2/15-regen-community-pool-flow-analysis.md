#### Protocol Mechanics

**Community Pool Module**:

```go
type CommunityPool struct {
    Balance    sdk.Coins
    BlockReward sdk.Dec   // 2% of block rewards
}
```

The pool operates as a specialized module account within the Cosmos SDK distribution module. Block rewards flow automatically based on chain parameters, with no manual intervention required.

**Proposal Submission Flow**:

1. Proposer creates JSON proposal file
2. Submits with 2,000 REGEN deposit
3. Enters 14-day deposit period
4. Moves to 7-day voting if deposit met
5. Executes automatically if passed

**Smart Contract/Module Design**:

Key functions:

- `SubmitProposal()`: Initiates governance process
- `Deposit()`: Adds to proposal deposit
- `Vote()`: Records stakeholder preference
- `TallyResult()`: Calculates final outcome

**Voting Power Calculation**:

```
VotingPower = StakedTokens × ValidatorStatus
where ValidatorStatus = 1 if in active set, 0 otherwise
```

**Quorum Calculation**:

```
QuorumMet = (YesVotes + NoVotes + NoWithVetoVotes + AbstainVotes) / TotalStakedTokens ≥ 0.4

```
Passed = QuorumMet AND
         (YesVotes / (YesVotes + NoVotes + NoWithVetoVotes)) > 0.5 AND
         (NoWithVetoVotes / (YesVotes + NoVotes + NoWithVetoVotes)) < 0.334

## 7. TERMINOLOGY GLOSSARY

### 7.1 Technical Terms

**Block Reward**: Newly minted REGEN tokens distributed each block (~6-7 seconds) to validators and delegators. 2% flows to community pool. Example: "Block rewards ensure sustainable pool funding."

**Community Pool**: Special module account receiving 2% of all block rewards, controlled by governance. Context: Primary funding source for ecosystem development. Related: Treasury, DAO funds.

**Community Spend Proposal**: Governance proposal type requesting token distribution from community pool. Must specify amount, recipient, and purpose. Example: "The community spend proposal requested 400,000 REGEN for development."

**Deposit Period**: 14-day window for proposal to accumulate 2,000 REGEN deposit before moving to vote. Context: Prevents spam while allowing community coordination. Related: MinDeposit, MaxDepositPeriod.

**Quorum**: Minimum 40% of staked tokens must participate in vote for validity. Context: Ensures sufficient participation for legitimacy. Example: "The proposal exceeded quorum with 52% participation."

**Tally**: Final calculation of Yes, No, NoWithVeto, and Abstain votes. Occurs at voting period end. Context: Determines proposal outcome. Related: Threshold, VetoThreshold.

**uregen**: Smallest unit of REGEN token (1 REGEN = 1,000,000 uregen). Used in technical specifications. Context: Precision unit for programming. Example: "The parameter requires 2000000000 uregen."

**Validator Set**: Top 75 validators by stake who produce blocks and vote on behalf of delegators. Context: Only active validator votes count toward proposals. Related: Delegation, Staking.

**Veto Threshold**: If 33.4% vote NoWithVeto, proposal fails and deposit burns. Context: Prevents malicious proposals. Example: "The controversial proposal risked hitting veto threshold."

**Voting Period**: 7-day window for staked token holders to vote on proposals. Reduced from 14 days via governance. Context: Balances participation with efficiency. Related: VotingParams.

### 7.2 Regen-Specific Nomenclature

**Climate Wiki**: First major community pool funding recipient, creating knowledge commons for climate action. 400,000 REGEN allocated via Proposal #12.

**Cosmos ZERO**: Initiative to offset carbon footprint of Regen Network validators through community pool funding. Achieved via Proposals #23, #25, and #30.

**csDAO**: Community Staking DAO - Foundation program delegating tokens to underrepresented groups. Separate from community pool but complementary governance mechanism.

**ECO Credits**: Ecological credits issued on Regen Network, distinct from governance tokens. Community pool has funded credit purchases for environmental impact.

**Regen Foundation**: 501(c)(3) managing 30% of token supply for community benefit. Separate from community pool but coordinates on ecosystem funding.

**RND**: Regen Network Development, Inc. - Public benefit corporation developing core protocol. Does not control community pool.

**Stakeholder Voting**: Inclusive governance allowing any amount of staked REGEN to participate. No minimum threshold unlike many DAOs.

**Calculate Pool Growth**:

```python
# Analysis script for pool growth projection
import pandas as pd
import numpy as np

def calculate_pool_growth(
    initial_balance=2000000,
    block_reward_rate=0.02,
    annual_inflation=0.15,
    spending_rate=0.25,
    years=5
):
    """Project community pool balance over time"""

    results = []
    balance = initial_balance

    for year in range(years):
        # Calculate annual inflow
        annual_blocks = 365 * 24 * 60 * 60 / 6.7  # ~4.7M blocks
        block_rewards = annual_blocks * (balance * annual_inflation / annual_blocks)
        pool_inflow = block_rewards * block_reward_rate

        # Apply spending
        spending = balance * spending_rate if year > 0 else 0

        # Update balance
        balance = balance + pool_inflow - spending

        results.append({
            'year': year,
            'balance': balance,
            'inflow': pool_inflow,
            'spending': spending
        })

    return pd.DataFrame(results)

# Run projection
df = calculate_pool_growth()
print(df)
```

### 11.2 Calculation Methodologies

**Annual Pool Growth Formula**:

```
Annual_Growth = Σ(Block_Rewards × Community_Tax_Rate)
where:
- Block_Rewards = (Total_Staked × Inflation_Rate) / Blocks_Per_Year
- Community_Tax_Rate = 0.02 (2%)
- Blocks_Per_Year ≈ 4,730,400 (at 6.7s block time)
- Inflation_Rate = variable 3.5-10%
```

**Voting Power Calculation**:

```
Individual_Voting_Power = Staked_Tokens × Validator_Weight
where:
- Validator_Weight = 1 if validator in active set (top 75)
- Validator_Weight = 0 if validator inactive
- Delegator can override validator vote
```

**Assumptions Made**:

1. Block time remains constant at 6.7 seconds
2. Inflation parameters follow current bounds
3. Validator set size remains at 75
4. No significant parameter changes via governance

**Margin of Error**:

- Block reward calculations: ±2% due to variable block time
- USD values: ±10% due to price volatility
- Participation rates: ±5% based on sampling
